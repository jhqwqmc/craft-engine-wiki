## 简介

每件家具至少要定义一个变体。变体决定了家具长什么样、占据多大空间。

:::info

变体名可以随便起，系统本身不做限制。你常见的 `ground`、`ceiling`、`wall` 只是默认 `furniture_item` 行为的约定：它根据点击面选变体——顶面 → `ground`，底面 → `ceiling`，侧面 → `wall`。

完全可以用其他名字。比如给电池家具定义 `charged`、`half_charged`、`empty` 三个变体，由 API 在电量变化时切换。光传感器也可以在放置时根据当前时间选 `day` 或 `night`。

:::

:::warning

默认的 `furniture_item` 只认 `ground`、`ceiling`、`wall` 三个名字。想用自定义变体名，要么注册自己的物品行为，要么放置后用 [`set_furniture_variant`](../reference/events.mdx#set_furniture_variant) 切过去。

:::

以下是一个两格高落地灯的完整变体示例：

```yaml
furniture:
  default:standing_lamp:
    variants:
      ground:
        # 掉落物偏移，防止掉落物生成在方块内部（默认: 0,0,0）
        loot_spawn_offset: 0,0.5,0

        # 展示元素——家具的外观
        elements:
          - type: item_display
            item: default:standing_lamp

        # 家具碰撞箱——家具如何占据空间以及如何与玩家交互
        hitboxes:
          - type: interaction
            width: 1
            height: 2
            blocks_building: true
            interactive: true
```

## 元素

**元素**是组成家具的各个独立部件。大多数家具一个元素就够了。需要复杂设计时，可以组合多个元素。比如一个全息投影仪可以拆成底座和投影两个元素：底座固定朝向，投影始终面朝玩家。

### 物品展示实体

https://minecraft.wiki/w/Display

**必填参数**

```yaml
type: item_display
item: default:bench
```

**可选参数**

```yaml
display_transform: none # none / third_person_left_hand / third_person_right_hand
                        # first_person_left_hand / first_person_right_hand
                        # head / gui / ground / fixed
billboard: fixed  # fixed / vertical / horizontal / center
position: 0.5,0,0
translation: 0,0.5,0
pitch: 0.0
yaw: 0.0
scale: 1 # scale: 1,2,1
glow_color: 255,255,255        # RGB
shadow_radius: 0
shadow_strength: 1
brightness:
  block_light: 15
  sky_light: 15
view_range: 1.0
# rotation 支持 3 种格式
rotation: 45  # 绕 Y 轴
rotation: 45,45,0  # 欧拉角，参见下方旋转工具
rotation: 0,0,0.7071,0.7071  # 四元数，参见下方旋转工具
# 从源物品提取颜色。根据元素所用的物品选一种。
tint_source: 
  type: dyed_color             # dyed_color / map_color / firework_explosion / potion_contents
```

:::warning
`position` 改的是实体坐标，`translation` 是在坐标不动的前提下偏移实体位置。

`wall` 和 `ceiling` 变体如果 `position` 留在 `0,0,0`，实体会发黑。给 `position` 一个微小偏移，再用反向的 `translation` 拉回原位：

```yaml
wall:
  elements:
    - type: item_display
      item: default:my_furniture
      position: 0,0,0.5        # 从墙面推出半格
      translation: 0,0,-0.5    # 拉回来，确保渲染位置正确
```

:::

### 交互实体

实体和玩家可以穿过。用于点击检测、座位、阻挡放置，不产生物理碰撞。

```yaml
type: interaction
position: 0,0,0
width: 1                         # 或使用 scale: 1,2 作为宽 × 高的简写
height: 2
interactive: true                # 玩家是否可以点击（默认: true）
blocks_building: true            # 是否阻止方块放置（默认: true）
can_use_item_on: true            # 放置的物品是否可以与其交互（默认: true）
can_be_hit_by_projectile: true   # （默认: true）
invisible: false                 # 是否通过 F3+B 可见（默认: false）
seats:
  - 0,0,-0.1 0
```

### 潜影贝

实体和玩家无法穿过。

```yaml
type: shulker
position: 1,0,0
scale: 1                          # 大小倍率（默认: 1）
peek: 0                           # 外壳打开程度，0~100（默认: 0）
direction: UP                     # up / down / north / west / east / south
interaction_entity: true          # 额外生成交互实体，提升点击精度
interactive: true                 # （默认: true）
blocks_building: true             # （默认: true）
can_use_item_on: true             # （默认: true）
can_be_hit_by_projectile: true    # （默认: true）
invisible: false                  # （默认: false）
seats:
  - 1,0,-0.1 0
```

### 快乐恶魂

大型碰撞箱（scale=1 时占用 4×4 格）。`hard_collision` 为 true 时实体无法穿过。

```yaml
type: happy_ghast
position: 1,0,0
scale: 0.25                       # 大小倍率（默认: 1）
hard_collision: true              # 是否阻挡实体移动
blocks_building: true             # （默认: true）
can_use_item_on: true             # （默认: true）
can_be_hit_by_projectile: true    # （默认: true）
seats:
  - 1,0,-0.1 0
```

### 自定义

使用任意原版实体类型进行碰撞检测。默认使用 `slime`。

```yaml
type: custom
position: 1,0,0
scale: 5                          # 乘以实体的自然大小
entity_type: slime                # 任意原版实体 ID，如 minecraft:creeper
blocks_building: true             # （默认: true）
can_use_item_on: true             # （默认: true）
can_be_hit_by_projectile: true    # （默认: true）
seats:
  - 1,0,-0.1 0
```

:::info
座位格式为 `x,y,z`，后面可接空格和 yaw 角度。有 yaw 时身体旋转受限，不能 360° 转。不写则自由旋转。

一个碰撞箱可以容纳多个座位。如果不同碰撞箱的座位位置重叠，只有一个会生效。

```
0,0,0 0    # 身体旋转受限
0,0,0      # 身体可自由旋转
```

## 实体剔除

<Highlight color="#f2184eff">**付费版专属**</Highlight>

实体剔除会在玩家看不到家具时自动隐藏它，节省性能。默认开启。

为某个变体关闭：

```yaml
ground:
  entity_culling: false
```

精细调整：

```yaml
ground:
  entity_culling:
    aabb: -0.5,0,-0.5,0.5,2,0.5   # 手动指定包围盒（x1,y1,z1,x2,y2,z2），不填则自动计算
    aabb_expansion: 0.25           # 包围盒向外扩展量（默认: 0.25）
    view_distance: 64              # 超过此距离（格）则裁剪，-1 为无限
    ray_tracing: true              # 是否做射线追踪检测（默认: true）
```

## 外部模型

变体可以引用外部插件的模型（BetterModel、ModelEngine，或通过 API 注册的来源）。

```yaml
furniture:
  default:bench:
    variants:
      ground:
        blueprint: blueprint_id    # 外部插件的模型 ID
```

<Details summary="外部模型 vs. 元素——区别是什么">

作为**元素**（写在 `elements` 下）时，模型跑在假实体上——直接发包给客户端，按玩家独立生成和销毁，服务端不存在。因为是假的，你想挂多少个都行。

作为**外部模型**（变体级别的 `blueprint`）时，模型挂在家具真正的基实体上——服务端真实存在。因为绑定的是真实实体，每个变体最多挂一个。

</Details>
