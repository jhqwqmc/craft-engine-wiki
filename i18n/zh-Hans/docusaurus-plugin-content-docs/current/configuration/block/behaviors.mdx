## 简介

方块行为定义了方块独特的机制与物理特性——例如如何生长、下落、开启、蔓延，或如何与相邻方块互动。原版 Minecraft 将这些机制硬编码进特定方块中，CraftEngine 则允许你通过配置，把同样的机制指定到任意方块上。

某些行为依赖特定的方块属性才能工作，例如：

- [农作物](behaviors/crop_block.mdx)需要 `age` 属性来追踪生长阶段
- [门](behaviors/door_block.mdx)需要 `hinge` 属性来决定门轴位于哪一侧

在后续各个行为的页面中，你会看到类似下面的属性表，用于说明该行为支持的属性名称、类型以及是否必需。

| 属性名称        | 属性类型      | 是否必需 |
|-------------|-----------|------|
| facing      | direction | 是    |
| waterlogged | boolean   | 否    |

:::info

未来版本可能会支持自定义属性名称。目前，为了简化配置，属性名称是硬编码的。有关类型和保留名称的完整列表，请参考[ℹ️ 属性](states/properties.mdx)页面。

:::

## 指定行为

通过方块的 `behavior` 或 `behaviors` 选项来指定行为：

```yaml
blocks:
  default:fairy_flower:
    behavior:
      type: bush_block
      bottom_block_tags:
        - minecraft:dirt
        - minecraft:farmland
```

## 复合行为

在 `behavior` 或 `behaviors` 下列出多个行为，即可将它们组合成一个复合方块行为。下面的例子中制作了一个既像[混凝土粉末](behaviors/concrete_powder_block.mdx)（遇水凝固）又会[像沙子一样下落](behaviors/falling_block.mdx)的方块：

```yaml
blocks:
  default:gunpowder_block:
    behaviors:
      - type: concrete_powder_block
        solid_block: default:solid_gunpowder_block
      - type: falling_block
```

:::warning

**组合多个行为时，请注意：**

- **确保包含所有必需属性。** 每个行为都会列出其所需的属性。组合多个行为时，方块必须满足所有行为的属性要求，否则整个方块将无法正常使用。
- **部分行为之间可能存在冲突。** 某些行为组合后可能出现非预期表现。如果遇到异常情况，请联系技术支持，我们会协助排查并尽可能解决问题。
:::
