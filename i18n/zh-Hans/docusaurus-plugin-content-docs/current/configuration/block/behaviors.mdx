## 简介

方块行为定义了方块独特的机制与物理特性——例如如何生长、下落、开启、蔓延，或如何与相邻方块互动。原版 Minecraft 将这些机制硬编码进特定方块中；Craft Engine 则允许你通过配置，把同样的机制挂载到任意方块上。

某些行为依赖特定的方块属性才能工作，例如：

- [作物方块](behaviors/crop_block.mdx)需要 `age` 属性来追踪生长阶段
- [门](behaviors/door_block.mdx)需要 `hinge` 属性来决定铰链位于哪一侧

在后续的每个行为页面开头，你都会看到类似下面这样的属性表，列出该行为所需的属性名称、类型，以及是否为必需。

| 属性名称    | 属性类型   | 是否必需 |
|---------|--------|-------|
| facing  | direction | 是    |
| waterlogged | boolean | 否    |

:::info

后续版本可能会支持自定义属性名。目前属性名采用硬编码，以保持配置简单。完整的类型与保留名称列表请见[属性](states/properties.mdx)页面。

:::

## 指定行为

通过方块的 `behavior` 或 `behaviors` 字段来挂载行为：

- 方块只有**一个**行为时，使用 **`behavior`**。
- 方块组合**多个**行为时，使用 **`behaviors`**（列表）。

```yaml
blocks:
  default:fairy_flower:
    behavior:
      type: bush_block
      bottom_block_tags:
        - minecraft:dirt
        - minecraft:farmland
```

## 复合行为

在 `behaviors` 下列出多个条目，即可将它们组合成一个复合方块。下面的例子让一个方块既像[混凝土粉末](behaviors/concrete_powder_block.mdx)（接触水后凝固）又会[像沙子一样下落](behaviors/falling_block.mdx)：

```yaml
blocks:
  default:gunpowder_block:
    behaviors:
      - type: concrete_powder_block
        solid_block: default:solid_gunpowder_block
      - type: falling_block
```

:::warning

**组合行为时请注意：**

- **合并所需属性。** 每个行为都列出了它需要的属性；组合时你的方块必须满足全部属性，否则某些行为将无法正常工作。
- **某些组合会冲突。** 部分行为之间并不兼容，可能出现意外表现。如果遇到问题，请联系技术支持，我们会协助解决。
- **不可重复。** 同一方块上不能多次列出同一种行为类型——这必然会导致冲突。
:::
